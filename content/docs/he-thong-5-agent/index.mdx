## Tư duy thiết kế

**5 agent. Ít mà sâu.**

Mỗi agent là một năng lực thật — không thêm cho có. Đủ để một mình vẫn có cảm giác như đang dẫn cả một team.

Không xây agent theo kiểu “AI biết mọi thứ”. Mỗi agent cần có:

- Một vai trò rõ.
- Một nhóm việc lặp rõ.
- Input/output rõ.
- Tool cần dùng.
- Skill cần có.
- Profile Hermes riêng.
- Quyền hạn và điểm duyệt.
- Cách báo cáo về Hermes Core.

## Cách viết mỗi case setup

Mỗi agent sẽ là một case setup riêng theo format:

1. **Vấn đề kinh doanh** — agent này giải quyết bottleneck gì?
2. **Vai trò agent** — agent là ai, thuộc phòng ban nào?
3. **Profile Hermes đề xuất** — tên profile, scope, model phù hợp.
4. **Input cần có** — dữ liệu, file, sheet, template, quyền truy cập.
5. **Output mong muốn** — kết quả đúng trông như thế nào?
6. **Tool cần kết nối** — Telegram, file, Gmail, Sheets, browser, image/video.
7. **Skill cần tạo** — checklist/sop nào agent cần nhớ?
8. **Automation gợi ý** — cron/webhook nào nên chạy?
9. **Approval rule** — việc nào tự làm, việc nào CEO duyệt?
10. **Dashboard/log** — agent báo cáo về đâu?
11. **Setup checklist** — từng bước triển khai profile.
12. **Bài test đầu tiên** — lệnh mẫu để kiểm tra agent chạy đúng.
