# KADA Vietnam — Curriculum Overview

> **Korea-ASEAN Digital Academy (KADA), Vietnam cohort**
>
> **5 weeks · 23 teaching days · approximately 133 contact hours · 5 modules · 40–45 students per cohort**

## Graduate profile

A KADA graduate can take a product idea from concept to a working, deployed application and present it as portfolio-ready work.

Across five weeks, students move through generative AI, product design, front-end development, backend and database engineering, mobile, data, machine learning, Git, DevOps, and an integrated capstone.

By graduation, each student should be able to:

- Build, judge, and defend their work — not merely prompt AI.
- Direct and verify AI-assisted development.
- Connect a React front-end to a .NET backend and database.
- Understand data, machine-learning, Git, CI/CD, and deployment fundamentals.
- Ship and publicly present an end-to-end product.
- Produce a GitHub README, CV entry, project description, promo video, and marketing images.

## How the five modules connect

The modules are one continuous learning arc around a single product idea:

> **Product idea → React front-end → .NET API + database → data & deployment → shipped capstone**

Students do not leave with five disconnected exercises. Each module hands work forward to the next module and strengthens the same product.

## Teaching principles

1. **Manual first, then AI** — students understand the mechanics before using AI to multiply their work.
2. **Critical questioning and coaching** — students justify choices and defend results.
3. **Verification as a core skill** — source checks, code inspection, execution-plan reading, sanity checks, and hallucination checklists are explicit learning outcomes.
4. **One product, continuous handoff** — every module produces an artifact used by the next module.
5. **Hands-on output over written exams** — assessment emphasizes demonstrated work and project quality.

## Teacher contribution guide

The lesson pages in Modules 2–4 are intentionally **outline scaffolds**, not finished teaching material. The assigned Teacher owns the final content, examples, labs, references, and assessment details.

For every lesson, please complete:

- Learning objectives and key concepts.
- Real-world examples connected to the shared product.
- Hands-on lab with clear input, task, checkpoint, and output.
- Verification criteria and common mistakes.
- Homework and handoff to the next lesson/module.
- References, tools, setup instructions, and rubric notes.

Please preserve the shared lesson structure so the final handbook feels consistent for students and other Teachers.
