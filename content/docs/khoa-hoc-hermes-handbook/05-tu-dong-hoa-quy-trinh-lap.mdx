## Không phải việc lặp nào cũng nên tự động hóa

Trước khi tạo cron/webhook, phân loại việc theo độ rõ quy trình và mức rủi ro.

Mỗi chấm là một việc. Bấm vào chấm để xem nên xếp nó vào nhóm nào và vì sao. Trục dọc là **độ rõ quy trình & đầu ra**, trục ngang là **mức rủi ro**.

<AutomationMatrix05 />

<Callout>
  <p>Việc rủi ro thấp + đầu ra rõ → có thể tự chạy. Việc gửi ra ngoài / sửa dữ liệu / phát sinh chi phí → cần duyệt. Việc cần phán đoán kinh doanh cao → chỉ nhắc hoặc hỗ trợ.</p>
</Callout>

## Thiết kế automation trước khi bật

Đừng tạo automation bằng một câu mơ hồ như “mỗi sáng báo cáo cho anh”.

Câu đó thiếu quá nhiều thứ: báo cáo gì, lấy dữ liệu ở đâu, format ra sao, khi nào cảnh báo, có được sửa dữ liệu không? Một automation tốt cần đủ **7 phần** — bấm từng phần để xem câu hỏi cần trả lời:

## Log: để biết agent đã làm gì

Một hệ thống tự động hóa cần có log. Không cần phức tạp.

Ban đầu chỉ cần ghi: automation chạy lúc nào, đọc input gì, tạo output gì, có lỗi không, có cần CEO duyệt gì không. Bấm một câu hỏi bên phải để lọc log:

<LogFeed05 />

Log giúp CEO trả lời:

> “Hermes đã làm gì rồi?”  
> “Có lỗi ở đâu không?”  
> “Việc nào đang chờ mình duyệt?”

## Hermes Operator Challenge

**Mục tiêu:** thiết kế automation đầu tiên theo lịch hoặc sự kiện, có trigger/output/escalation rõ.

### Output cần nộp

Tạo file:

```text

```

Brief này là bản thiết kế trước khi tạo cron/webhook thật.

|---|---|
| Tên automation |  |
| Trigger/lịch chạy |  |
| Input agent cần lấy |  |
| Output gửi CEO |  |
| Việc agent tự làm |  |
| Việc cần CEO duyệt |  |
| Escalation/báo gấp |  |
| Log lưu ở đâu |  |

### Prompt copy dùng ngay

```text
Hãy giúp anh thiết kế automation đầu tiên cho Hermes.

Việc lặp muốn tự động hóa:
[tên việc]

Mục tiêu kinh doanh:
[mục tiêu]

Tần suất/lịch mong muốn:
[hằng ngày/hằng tuần/khi có sự kiện]

Hãy trả về automation brief gồm:
- Trigger/lịch chạy
- Input cần lấy
- Quy trình xử lý
- Output gửi CEO dưới 10 dòng
- Việc được tự làm
- Việc cần hỏi CEO
- Rule escalation nếu có rủi ro
- Log cần lưu
- Bài test trước khi bật thật
```

### Tiêu chuẩn hoàn thành

- [ ] Chọn được 1 việc lặp phù hợp để tự động hóa.
- [ ] Có trigger rõ: lịch hoặc sự kiện.
- [ ] Output gửi CEO ngắn, dễ duyệt.
- [ ] Có rule escalation.
- [ ] Có log.
- [ ] Sẵn sàng tạo cron/webhook thử nghiệm.
