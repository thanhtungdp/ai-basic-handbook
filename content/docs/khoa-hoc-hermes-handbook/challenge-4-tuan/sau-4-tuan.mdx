## Tổng quan tuần 5–8

| Tuần | Mục tiêu | Output cuối tuần |
|---|---|---|
| Tuần 5 | Đo hiệu quả agent | Bảng ROI + số liệu tiết kiệm thời gian |

## Tuần 5 — Đo hiệu quả agent

**Câu hỏi:** Agent có thực sự tiết kiệm thời gian không?

### Việc cần làm

| # | Việc | Cách làm |
|---|---|---|
| 1 | Đếm giờ tiết kiệm | So sánh: việc cũ mất bao lâu vs agent làm mất bao lâu |
| 2 | Đếm số lần agent chạy | Check log / đếm output files |
| 3 | Đánh giá chất lượng | Review 5 output gần nhất — đạt mấy/5? |
| 4 | Hỏi: CEO có còn phải sửa? | Nếu sửa nhiều → skill cần cải thiện |
| 5 | Lập bảng ROI | Giờ tiết kiệm × giá giờ CEO = tiền tiết kiệm |

### Output cuối tuần 5

```markdown
# Agent ROI Report — Tháng 1

| Agent | Việc | Giờ tiết kiệm/tuần | Chất lượng (/5) | Cần cải thiện? |
|---|---|---|---|---|
| Sales | Báo giá | 4 giờ | 4/5 | Giảm lỗi placeholder |
| Content | Social post | 3 giờ | 5/5 | Không |
| Daily | Daily Brief | 2 giờ | 4/5 | Thêm số liệu công nợ |

Tổng tiết kiệm: 9 giờ/tuần ≈ 36 giờ/tháng
```

## Tuần 6 — Chuẩn hóa 3 skill tốt nhất

**Câu hỏi:** Skill nào đáng giữ, skill nào cần sửa?

### Việc cần làm

| # | Việc | Cách làm |
|---|---|---|
| 1 | Liệt kê tất cả skill đã tạo | Đếm từ folder skills |
| 2 | Review từng skill | Chạy thử 2 lần — xem output ổn không |
| 3 | Chọn top 3 skill | Theo: tần suất dùng + chất lượng + ROI |
| 4 | Tối ưu top 3 | Sửa pitfall, thêm checklist, cải thiện prompt |
| 5 | Sunset skill yếu | Xóa hoặc merge vào skill khác |

### Tiêu chí skill "đạt"

- [ ] Chạy 2 lần liên tiếp không lỗi
- [ ] Output đạt ≥ 4/5 tiêu chí checklist
- [ ] CEO không phải sửa > 10% output
- [ ] Có prompt mẫu rõ
- [ ] Có pitfall section

## Tuần 7 — Thêm automation thứ hai

**Câu hỏi:** Agent còn việc gì lặp mà chưa tự động?

### Việc cần làm

| # | Việc | Cách làm |
|---|---|---|
| 1 | Xem lại Repeated Work Map | Còn việc nào chưa agent hóa? |
| 2 | Chọn 1 việc tiếp theo | Ưu tiên: việc định kỳ + rủi ro thấp |
| 3 | Tạo skill | Theo Template SOP → Skill |
| 4 | Tạo cron/webhook | Đặt lịch hoặc trigger |
| 5 | Test 5 ngày liên tiếp | Nếu ổn → production |

### Gợi ý automation thứ hai

| Việc | Loại | Rủi ro |
|---|---|---|
| Weekly content plan | Cron thứ 2 hằng tuần | Thấp |
| Lead research | On-demand | Thấp |
| Follow-up reminder | Cron hằng ngày | Thấp |
| Cashflow brief | Cron thứ 2 sáng | Thấp |
| Client onboarding checklist | Webhook | Trung |

## Tuần 8 — Tách agent theo vai trò

**Câu hỏi:** Agent nào nên chuyên trách việc gì?

### Việc cần làm

| # | Việc | Cách làm |
|---|---|---|
| 1 | Vẽ agent org chart | Ai báo ai, agent nào làm việc gì |
| 2 | Tách profile | Mỗi agent 1 profile riêng: model, tools, memory |
| 3 | Ranh giới rõ | Agent A không làm việc của Agent B |
| 4 | Escalation rule | Khi nào agent báo CEO, khi nào báo agent khác |
| 5 | Log + dashboard | Mỗi agent có log riêng, CEO xem được |

### Agent org chart mẫu

```
CEO
├── Hermes Core (điều phối)
│   ├── Hermes Sales (báo giá, follow-up, pipeline)
│   ├── Hermes Content (social, blog, content plan)
│   ├── Hermes Ops (onboarding, CS, báo cáo)
│   └── Hermes Daily (cron, brief, công nợ)
```

### Ranh giới ví dụ

| Agent | Làm | Không làm |
|---|---|---|
| Sales | Báo giá, follow-up | Content, báo cáo vận hành |
| Content | Viết, lên lịch | Báo giá, email sales |
| Ops | Onboarding, CS | Báo giá, content |
| Daily | Cron, brief, công nợ | Tạo content, báo giá |

## Sau tháng 2 — Review định kỳ

Mỗi tháng, CEO nên:

| Review | Tần suất | Mục đích |
|---|---|---|
| ROI agent | Tháng | Đo giờ tiết kiệm, chất lượng |
| Skill audit | Tháng | Skill nào dùng nhiều, skill nào chết |
| Cost audit | Tháng | Chi phí AI có tăng quá nhanh? |
| Agent boundary | Quý | Agent nào quá tải? Cần tách? |
| New use case | Quý | Có việc mới lặp → agent hóa |
