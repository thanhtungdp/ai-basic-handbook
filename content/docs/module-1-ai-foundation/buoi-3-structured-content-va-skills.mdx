### 2. Topic Clustering — Biến tài liệu lộn xộn thành cấu trúc

Tài liệu thô (note họp, email thread, báo cáo rời rạc) → Claude có thể nhóm thành topic cluster có cấu trúc.

**Ví dụ:** 20 trang note họp → Claude nhóm thành:

| Cluster | Chủ đề | Entity | Metadata |
|---|---|---|---|
| Khách hàng | Tên, nhu cầu, trạng thái | customer_id, stage | last_contact, value |
| Sản phẩm | Tính năng, giá, roadmap | product_id, status | priority, eta |
| Vận hành | Quy trình, bottleneck | process_id, owner | frequency, risk |

<Callout variant="orange">
  <p><strong>Tại sao quan trọng?</strong> Structured input → AI output tốt hơn. Claude không phải đoán mò — nó có cấu trúc để bám theo.</p>
</Callout>

### 3. Search Engines vs Answer Engines

| | Search Engine (Google) | Answer Engine (Claude/ChatGPT) |
|---|---|---|
| **Output** | Danh sách link | Câu trả lời tổng hợp |
| **Bạn làm gì** | Click từng link, tự tổng hợp | Đọc câu trả lời, verify |
| **Khi nào dùng** | Cần tìm nguồn chính xác | Cần tổng hợp nhanh |
| **Rủi ro** | Tốn thời gian | Hallucination — cần kiểm chứng |

<Callout>
  <p><strong>Thực chiến:</strong> Dùng Answer Engine tổng hợp → Search Engine verify nguồn. Không dùng 1 cái thay cái còn lại.</p>
</Callout>

### Bước 5 — Thử Skill (15 phút)

1. Lưu Skill vào Claude Project (Custom Instructions hoặc file).
2. Gửi 1 input mới (không phải input đã build Skill).
3. Kiểm tra: Claude có gọi đúng format không? Output đúng chuẩn không?
