## Mục tiêu buổi học

Sau buổi này, mỗi học viên có thể:

- Xây prompt template dùng lại được cho quy trình lặp.
- Sketch 1 workflow input → AI processing → structured output.
- Phân biệt 3 mô hình: Claude (session), ChatGPT Work (background tasks), Hermes Agent (always-on).
- Biết khi nào chọn mô hình nào cho use case của mình.

## Kiến thức trọng tâm

### 1. Prompt Template — Từ 1 prompt thành quy trình

Prompt 1 lần → Prompt template = **biến số + cấu trúc cố định**.

<Callout>
  <p><strong>Khác biệt:</strong> Prompt thường viết cho 1 việc cụ thể. Prompt template có "khe" để điền input mới — dùng lại được mọi lần.</p>
</Callout>

**Ví dụ:**

| | Prompt 1 lần | Prompt Template |
|---|---|---|
| Input | "Tóm tắt email này: [paste email]" | "Tóm tắt email: `{{EMAIL_CONTENT}}`" |
| Dùng lại | Không — viết lại | Có — thay `{{EMAIL_CONTENT}}` mỗi lần |
| Mở rộng | Khó | Dễ — thêm biến `{{OUTPUT_FORMAT}}`, `{{TONE}}` |

### 3. Ba mô hình làm việc với AI

| | Claude | ChatGPT Work | Hermes Agent |
|---|---|---|---|
| **Bản chất** | Session-based assistant | Background tasks trong session | Always-on agent 24/7 |
| **Chạy khi nào** | Khi bạn mở chat | Khi bạn giao task — chạy nền | Luôn luôn — kể cả khi bạn ngủ |
| **Kênh giao tiếp** | Web interface | Web interface + API | Telegram, CLI, API, webhook |
| **Tự ghi nhớ** | Project knowledge | Project / Custom Instructions | Memory + Skill + Session Search |
| **Chạy định kỳ** | Không | Có giới hạn | Cron + Webhook — đầy đủ |
| **Phù hợp** | Phân tích, viết, tư vấn | Research, tạo file, task nền | Tự động hóa vận hành hằng ngày |

### 4. Khi nào chọn mô hình nào?

| Use case | Mô hình | Lý do |
|---|---|---|
| Viết email quan trọng, cần suy nghĩ | Claude | Chất lượng cao, tương tác trực tiếp |
| Tìm dữ liệu thị trường, crawl web | ChatGPT Work | Web search real-time, có nguồn |
| Mỗi sáng tóm tắt email + gửi briefing | Hermes Agent | Cron 7:30 tự chạy, không cần mở |
| Phân tích báo cáo dài 50 trang | Claude | Context window lớn, phân tích sâu |
| Theo dõi giá đối thủ mỗi giờ | Hermes Agent | Cron mỗi giờ, alert qua Telegram |
| Tạo presentation từ outline | ChatGPT Work | Tạo file nền, xong báo |

### Bước 3 — Class Debate: Claude vs ChatGPT Work vs Hermes Agent (20 phút)

Chia 3 nhóm, mỗi nhóm bảo vệ 1 mô hình:

| Nhóm | Bảo vệ | Lập luận chính |
|---|---|---|
| A | Claude | Chất lượng cao, tương tác trực tiếp, phân tích sâu |
| B | ChatGPT Work | Web search real-time, tạo file, chạy nền |
| C | Hermes Agent | Always-on 24/7, cron + Telegram, tự ghi nhớ |

Mỗi nhóm có 5 phút thuyết trình + 5 phút phản biện.

**Câu hỏi tranh luận:**
1. Nếu bạn là CEO 1 người, mô hình nào tiết kiệm thời gian nhất?
2. Việc nào KHÔNG nên tự động hóa?
3. Khi nào agent cần hỏi CEO trước khi hành động?

### Bước 4 — No-Code Automation Demo (20 phút)

Quan sát instructor demo 1 automation đơn giản:

```
Trigger: Mỗi sáng 7:30
  → Action: Hermes Agent chạy Skill tóm tắt email
  → Output: Gửi briefing qua Telegram
  → CEO đọc, duyệt, reply
```

Thảo luận: quy trình nào của bạn có thể tự động hóa tương tự?
