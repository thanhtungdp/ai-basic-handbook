## Tổng quan 6 template

| # | Template | Dùng để làm gì | Khi nào dùng |
|---|---|---|---|
| 1 | Repeated Work Map | Liệt kê việc lặp, chấm điểm ROI | Tuần 1 — Ngày 1–3 |
| 2 | Agent Brief | Mô tả agent cần làm gì, phạm vi, công cụ | Tuần 1–2 — trước khi tạo agent |
| 3 | SOP → Skill | Biến quy trình thủ công thành skill Hermes tự chạy | Tuần 2 — Ngày 6–10 |
| 4 | Approval Rule | Quy định việc nào agent tự làm / việc nào hỏi CEO | Tuần 2–3 — trước khi tự động hóa |
| 5 | Daily Brief | Báo cáo sáng hằng ngày từ agent | Tuần 3–4 — khi agent chạy hằng ngày |
| 6 | Weekly Review | Tổng kết tuần, đo hiệu quả agent | Tuần 4 — và mỗi tuần sau đó |

## Template 1 — Repeated Work Map

> **Mục tiêu:** Liệt kê 20–50 việc CEO đang lặp lại, chấm điểm ROI, chọn top 3 ưu tiên agent hóa.

### Cách dùng

1. Dành 30–45 phút ngồi liệt kê **tất cả** việc lặp lại trong tuần qua.
2. Chấm điểm mỗi việc theo 3 tiêu chí: Tần suất (1–5), Thời gian mỗi lần (1–5), Giá trị (1–5).
3. Sắp xếp theo tổng điểm, chọn top 3.

```markdown

| Hạng | Việc | Lý do chọn | Kỳ vọng tiết kiệm |
|---|---|---|---|
| 1 | | | ___ giờ/tuần |
```

## Template 3 — SOP → Skill

> **Mục tiêu:** Biến một quy trình thủ công thành skill Hermes có thể tái sử dụng, tự chạy.

## Template 4 — Approval Rule

> **Mục tiêu:** Quy định rõ việc nào agent tự làm, việc nào cần CEO duyệt trước/sau.

### Cách dùng

1. Liệt kê tất cả hành động agent có thể thực hiện.
2. Phân loại: Tự làm / Báo sau / Hỏi trước.
3. Áp dụng vào cron/skill/profile.

### Template copy-paste

```markdown
# Approval Rule: [Tên Agent / Toàn bộ đội agent]

## Nguyên tắc chung
- Agent tự làm khi: rủi ro thấp + đã có SOP + output có checklist
- Agent hỏi CEO khi: rủi ro cao / chưa có SOP / ảnh hưởng tiền/khách hàng
- Agent báo CEO sau khi xong khi: việc đã ổn định, CEO chỉ cần biết

## Bảng phân loại

| # | Hành động | Mức rủi ro | Chế độ | Ghi chú |
|---|---|---|---|---|
| 1 | Tạo báo giá nội bộ draft | Thấp | Tự làm | CEO duyệt trước khi gửi |
| 2 | Gửi báo giá cho khách | Cao | Hỏi trước | CEO must approve |
| 3 | Tạo email follow-up | Thấp | Tự làm | Gửi draft qua Telegram |
| 4 | Gửi email cho khách | Trung | Báo sau | Gửi cc CEO |
| 5 | Tạo content / đăng social | Thấp | Tự làm | Theo brand voice |
| 6 | Tạo báo cáo sáng | Thấp | Tự làm | Gửi 8h mỗi ngày |
| 7 | Truy cập dữ liệu khách hàng | Trung | Chỉ đọc | Không ghi/xóa |
| 8 | Sửa/xóa file | Cao | Hỏi trước | Backup trước |
| 9 | Gửi tiền / thanh toán | Rất cao | Hỏi trước | Luôn cần CEO |
| 10 | Gọi API ngoài (SendGrid, SePay...) | Trung | Báo sau | Log lại mọi call |

## Ký hiệu chế độ
- **Tự làm**: Agent thực hiện, không cần hỏi. CEO có thể review sau.
- **Báo sau**: Agent thực hiện, gửi kết quả cho CEO ngay sau khi xong.
- **Hỏi trước**: Agent chuẩn bị, gửi draft, chờ CEO duyệt mới thực hiện.

## Khi nào nâng cấp / hạ cấp
- Việc đang "Hỏi trước" → nếu 5 lần liên tiếp CEO approve không sửa → hạ xuống "Báo sau"
- Việc đang "Báo sau" → nếu 3 lần liên tiếp OK → hạ xuống "Tự làm"
- Việc đang "Tự làm" → nếu 2 lần liên tiếp lỗi/sai → nâng lên "Hỏi trước"
```

## Template 5 — Daily Brief

> **Mục tiêu:** Agent gửi CEO 1 báo cáo ngắn mỗi sáng — biết tình hình trong 30 giây.

### Cách dùng

1. Đặt cron chạy 8h sáng mỗi ngày.
2. Agent thu thập dữ liệu → gửi theo format dưới.
3. CEO đọc 30 giây, biết cần làm gì hôm nay.

### Template copy-paste

```markdown
# Daily Brief — ___/___/___

## 1. Tổng quan (2 dòng)
- Hôm nay cần chú ý: ___
- Trạng thái chung: Xanh / Vàng / Đỏ

## 2. Việc cần CEO xử lý hôm nay
| # | Việc | Khẩn cấp? | Deadline | Ghi chú |
|---|---|---|---|---|
| 1 | | ⬜/🟡/🔴 | | |
| 2 | | ⬜/🟡/🔴 | | |
| 3 | | ⬜/🟡/🔴 | | |

## 3. Agent đã làm hôm qua
| Agent | Việc | Kết quả | Cần CEO xem? |
|---|---|---|---|
| Sales | | ✅/⚠️/❌ | |
| Content | | ✅/⚠️/❌ | |
| Ops | | ✅/⚠️/❌ | |

## 4. Số liệu nhanh
- Lead mới: ___
- Deal đang mở: ___
- Công nợ quá hạn: ___
- Content đã đăng: ___

## 5. Cảnh báo / Rủi ro
- ⚠️ ___
- ⚠️ ___

## 6. Gợi ý hành động
1. ___
2. ___
```

### Lệnh cron mẫu

```text
Mỗi ngày 8h sáng, gửi Daily Brief theo template:
- Thu thập: lead mới, deal đang mở, công nợ quá hạn, content đã đăng
- Gửi qua Telegram
- Nếu có cảnh báo → đánh dấu 🔴
- Nếu không có gì cần CEO → ghi "Hôm nay yên tâm"
```

## Template 6 — Weekly Review

> **Mục tiêu:** Tổng kết tuần, đo hiệu quả agent, quyết định nâng cấp/thu hẹp phạm vi.

### Cách dùng

1. Cuối tuần (thứ 6 hoặc CN), agent tự tổng hợp.
2. CEO xem 5 phút, quyết định điều chỉnh tuần sau.
3. Lần sau agent tự điền dựa trên dữ liệu tuần.

### Template copy-paste

```markdown
# Weekly Review — Tuần ___ (___/___ đến ___/___)

## 1. Tổng quan tuần
- Điểm tuần (1–10): ___
- 1 dòng tóm tắt: ___

## 2. Agent performance

| Content | | | % | |
| Ops | | | % | |

## 3. Tiết kiệm thời gian
- Giờ tiết kiệm tuần này: ___ giờ
- Giờ tiết kiệm tích lũy: ___ giờ
- Ước tính tiết kiệm/tháng: ___ giờ

## 4. Việc CEO vẫn phải tự làm (chưa agent hóa)
```

## Cách dùng bộ template này trong 4 tuần

| Tuần | Template dùng | Output cuối tuần |
|---|---|---|
| Tuần 1 | Template 1 + 2 | Top 3 việc + 1 Agent Brief hoàn chỉnh |
| Tuần 2 | Template 2 + 3 + 4 | 1 Skill chạy được + Approval Rule |
| Tuần 3 | Template 3 + 4 + 5 | 1 Cron Daily Brief chạy hằng ngày |
| Tuần 4 | Template 5 + 6 | 1–2 agent chạy thật + Weekly Review đầu tiên |

## Lệnh giao Hermes tạo template

Nếu CEO muốn Hermes tự điền template dựa trên dữ liệu sẵn có:

```text
Dựa trên [thông tin doanh nghiệp / danh sách việc lặp / dữ liệu tuần qua], hãy điền Template [số] và gửi cho tôi kết quả. Nếu thiếu thông tin, hỏi tối đa 3 câu.
```

---

> **Nguyên tắc:** Template là khung. Agent là người điền. CEO là người duyệt.  
> Đừng cố điền hoàn hảo ngay — điền nhanh, chạy thử, sửa sau.
